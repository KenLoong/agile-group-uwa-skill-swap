# =============================================================================
# UWA Skill-Swap — production / runtime dependencies
# =============================================================================
#
# This file pins direct dependencies with explicit upper bounds so that
# `pip install -r requirements.txt` stays reproducible across machines and CI.
#
# Policy (agreed by the team, 2026):
#   - Use semantic-version caps: next major (x.y) is excluded unless vetted.
#   - Transitive dependencies are left to pip's resolver unless one breaks us;
#     then we add an explicit line here with a comment and issue link.
#   - When upgrading Flask, re-run the full test suite and smoke-test auth flows.
#   - Python baseline: 3.10+ (3.11 recommended in README).
#
# How to add a new package:
#   1) Add a line: packagename>=LATEST_TESTED,<NEXT_MAJOR
#   2) Note the ticket in a comment
#   3) Update docs/dependencies.md in the same PR if behaviour changes
#
# How to refresh pins safely:
#   pip install -r requirements.txt
#   pip list --outdated
#   # bump one package at a time, run tests, commit
#
# See also: requirements-dev.txt for Selenium, linters, and local-only tools.
# =============================================================================
